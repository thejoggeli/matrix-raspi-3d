Joerg_Koehler:

ut I read in the forum that juce needs a desktop environment to display an UI.

You do not need a desktop environment to run JUCE but you do need X11. No matter what type of graphics you are using, you need some kind of display backend (even for OpenGL or OpenGL ES) to handle user input (mouse, keyboard, etc.) and events. X11 is the most popular display backend and used by all major linux desktops but you don‚Äôt need to install a desktop environment to be able to use X11. There are other display backends such as wayland. Wayland is up-and-coming and a lot more light-weight than X11 but not supported by JUCE yet.
 
 

Joerg_Koehler:

Is OpenGl the key for my approach?

OpenGL is just a graphics accelerator. If your graphics are too slow then you should consider using OpenGL. But you still need some kind of display backend to actually show the graphics and process the input. OpenGL ES does allow you to display graphics without running a display backend but you still need something that would handle events and user input.

 

Joerg_Koehler:

Can I turn my ‚Äústandard‚Äù application into an OpenGL app without hassle?

As mentioned above, if your using JUCE you will need to run your app under X11. If your graphics are too slow, you can speed them up with OpenGL. This is very easy to do with JUCE. Simply attach an OpenGLContext 7 to your top-level component.
